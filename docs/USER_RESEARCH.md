# 사용자 리서치 보고서

> WandR — 프리미엄 여행지 추천 대시보드
> 리서치 기간: 2026-03-01 ~ 2026-03-14
> 작성자: WandR 개발팀

---

## 1. 리서치 목적

국내 20~35세 청년층의 해외여행 정보 탐색 행동 패턴을 파악하고,
기존 여행 플랫폼의 핵심 페인포인트를 도출하여 WandR의 UX 방향을 정의한다.

## 3. 핵심 페인포인트

### 3-1. 정보 분산 문제 (Critical)

> "날씨, 투어, 현지 팁을 3~4개 앱에서 각각 찾아야 해요.
> 한 곳에서 다 볼 수 있으면 좋겠어요."
> — 인터뷰 참가자 A (29세, 직장인)

- **발생 빈도**: 인터뷰 참가자 8명 중 7명(87.5%) 언급
- **영향**: 여행 계획 준비에 평균 2.3시간 소요

### 3-2. 모바일 UX 불편 (High)

- 기존 플랫폼의 텍스트 중심 UI → 모바일에서 가독성 저하
- 터치 영역이 작아 의도치 않은 탭 발생 (설문 응답자 61%)
- 가로 스크롤 콘텐츠의 부자연스러운 스크롤링

### 3-3. 신뢰도 부족 (Medium)

- 최신 정보 업데이트 시점 불명확
- 날씨 예보의 현지 정확도에 대한 불신
- 협찬 리뷰와 실제 리뷰 구분 불가

### 3-4. 비주얼 품질 낮음 (Medium)

> "유튜브나 인스타로 먼저 목적지를 결정하는데,
> 여행 앱은 왜 이렇게 촌스럽죠?"
> — 인터뷰 참가자 C (26세, 크리에이터)

- 여행지 사진 품질과 레이아웃에 대한 낮은 만족도
- 다크모드 미지원 (야간 탐색 시 눈부심)

## 4. 사용자 페르소나

### Persona A — "계획형 탐험가" 김지민 (28세)

```
직업: IT 스타트업 디자이너
여행 빈도: 연 2~3회 해외
여행 스타일: 3~4개월 전 계획, 스프레드시트로 일정 정리
페인포인트: 여러 탭을 열어두고 정보를 복사-붙여넣기
핵심 니즈: 여행지 비교, 날씨 확인, 투어 예약을 한 화면에서
```

### Persona B — "즉흥형 여행자" 박수진 (31세)

```
직업: 마케터
여행 빈도: 연 4~5회, 짧은 주말 여행 포함
여행 스타일: 출발 1~2주 전 결정, 인스타그램 + 유튜브 참고
페인포인트: 예쁜 시각 자료가 없으면 흥미 잃음
핵심 니즈: 비주얼 중심의 여행지 탐색, 빠른 정보 파악
```

## 5. 주요 인사이트 요약

| 순위 | 인사이트 | WandR 해결책 |
|------|----------|-------------|
| 1 | 정보 분산 → 탐색 피로 | 원스톱 대시보드 (날씨·투어·팁·브이로그 통합) |
| 2 | 모바일 UX 불편 | snap-x 스크롤, 대형 터치 타겟, 반응형 레이아웃 |
| 3 | 시각적 몰입감 부족 | 글래스모피즘 + Aurora 색상 + 풀스크린 히어로 |
| 4 | 할인/혜택 정보 부재 | 쿠폰 FAB + 쿠폰 모달 시스템 |
| 5 | 다크모드 미지원 | 다크모드 기본, 라이트모드 토글 |

## 7. 결론 및 WandR 설계 방향

1. **정보 통합**: 날씨·투어·브이로그·팁·쿠폰을 여행지 기준으로 한 화면에 표시
2. **모바일 퍼스트**: 375px 기준, 스냅 스크롤, 최소 44px 터치 타겟
3. **시각적 프리미엄**: 글래스모피즘 + 프리미엄 사진 + 매끄러운 애니메이션
4. **접근성**: WCAG 2.2 AA — 키보드, 스크린리더, 고대비 지원
5. **신뢰성**: 날씨 갱신 시간 표시, 출처 명확화, 에러 상태 명시
